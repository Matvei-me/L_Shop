# Документация к лабе (ветка docs)

По ТЗ после основной лабы коммиты с докой лежат в ветке **`docs`**, потом открываешь **Pull Request из `docs` в `main`** и **не мержишь** — препод смотрит diff.

## Swagger (API наглядно)

Поднял backend (`cd backend && npm run dev`), открываешь в браузере:

**http://localhost:3000/api-docs**

Там все ручки из `/api/...` и схемы тел запросов/ответов (типы как в OpenAPI).

Файл описания лежит в **`backend/openapi.json`** — если что-то поменяли в API, править нужно и код, и этот json, иначе дока разъедется с реальностью.

## JSDoc

**Backend:** комментарии повесил на то, что по сути утилиты — чтение/запись json, jwt-хелперы, middleware ошибки/авторизации, плюс пара функций в сервисах товаров.

**Frontend:** общие штуки — `utils/html.ts` (escape), обёртка `fetchApi` в `api.ts`, роутер в `router.ts`. Не стал расписывать каждый рендер страницы — там и так всё в одном файле.

## Кнопки на фронте

Вместо отдельных стилей для каждой кнопки сделал базовый класс **`btn`** и два варианта **`btn--primary`** / **`btn--danger`** (+ **`btn--small`** где узкая кнопка). Цвета простые, без цирка.

## Если препод спросит «где типизация во фронте»

Ответы API уже описаны интерфейсами в **`frontend-spa/src/types.ts`**, плюс дженерик у `fetchApi<T>`.
